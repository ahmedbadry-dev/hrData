## 1. المقدمة

### 1.1 الغرض
تحدد هذه الوثيقة المتطلبات الكاملة لمنصة **كفؤ** (Kafoo')، وهي منصة ذكية لتقديم الوظائف بنظام SaaS تستهدف سوق العمل السعودي. تعتبر هذه الوثيقة المرجع الوحيد لفريق التطوير، حيث تغطي جميع الميزات، الشاشات، نقاط نهاية API، نماذج البيانات، وسلوك النظام.

### 1.2 النطاق
تمكن منصة كفؤ الباحثين عن عمل من تجاوز قنوات التقديم التقليدية عبر إرسال السير الذاتية وخطابات التغطية **مباشرة** إلى عناوين البريد الإلكتروني لمديري التوظيف (HR) باستخدام حساب **Gmail الخاص بالمستخدم**. يقوم النظام بأتمتة جمع بيانات الوظائف عبر زحف الويب (Scraping)، ويستخدم تقنيات الذكاء الاصطناعي (LLM) لتحويل بيانات HTML الخام إلى كائنات وظيفية منظمة.

### 1.3 الجمهور المستهدف
- مهندسو الواجهة الخلفية والأمامية
- مهندسو معمارية النظام
- مديرو المنتجات
- مهندسو ضمان الجودة (QA)
- أصحاب المصلحة والمستثمرون

### 1.4 التعريفات والمصطلحات

| المصطلح | التعريف |
|------|------------|
| SaaS | البرمجيات كخدمة (Software as a Service) |
| HR | مدير الموارد البشرية / مسؤول التوظيف |
| CV | السيرة الذاتية (Resume) |
| Auto-Apply | ميزة الإرسال الجماعي المؤتمت للبريد الإلكتروني |
| Scraper | بوت مؤتمت يجمع بيانات الوظائف من مصادر خارجية |
| Open Rate | نسبة البريد الإلكتروني المفتوح من قبل المستلمين |
| Tracking Pixel | صورة شفافة 1x1 تضمن في البريد لتتبع فتحه |
| Gmail API | واجهة برمجة تطبيقات جوجل الرسمية لإرسال البريد نيابة عن المستخدم |
| OAuth2 | بروتوكول التفويض المستخدم لربط حسابات Gmail |
| BullMQ | مكتبة لإدارة الطوابير (Queues) مبنية على Redis لتجدولة المهام |
| LLM | نموذج لغوي كبير — ذكاء اصطناعي لتحليل البيانات |

## 2. الوصف العام للنظام

### 2.1 منظور المنتج
منصة كفؤ هي تطبيق ويب SaaS يعمل بشكل مستقل بواجهتين رئيسيتين:
- **بوابة المستخدم** — للباحثين عن عمل
- **لوحة تحكم المسؤول** — لمديري المنصة

يرتبط النظام بعدة أنظمة خارجية:
1. **مصادر الوظائف** (مواقع الوظائف السعودية بالعربية والإنجليزية) — تُسحب آلياً.
2. **LLM API** — لتحويل HTML الخام إلى بيانات منظمة.
3. **Gmail API** — لإرسال البريد من حساب المستخدم.
4. **خدمة تتبع البريد** — عبر بكسلات التتبع.
5. **Redis + BullMQ** — لإدارة طوابير الإرسال والجدولة.

### 2.2 ملخص وظائف المنتج
- جمع بيانات الوظائف تلقائياً من كبرى مواقع الوظائف السعودية.
- تحليل HTML المدعوم بالذكاء الاصطناعي لاستخراج بيانات منظمة.
- إرسال السير الذاتية مباشرة إلى بريد مديري التوظيف.
- جدولة إرسال البريد الإلكتروني.
- تكامل Gmail OAuth2.
- تتبع معدل فتح البريد الإلكتروني.
- حفظ الوظائف وتتبع تاريخ التقديمات.

### 2.3 السوق المستهدف
- **الأساسي:** الباحثون عن عمل في المملكة العربية السعودية.
- **الثانوي:** المحترفون في منطقة الخليج الذين يستهدفون الشركات السعودية.

## 5. المتطلبات الوظيفية — بوابة المستخدم

### 5.1 الصفحة الرئيسية (Landing Page)
**الغرض:** تقديم المنصة، شرح القيمة المضافة، وتحويل الزوار إلى مستخدمين مسجلين.

**الأقسام:**
- **قسم الـ Hero:** يشرح فكرة التقديم المباشر مع أزرار دعوة للعمل (CTA).
- **الميزات:** مجانية الاستخدام، تواصل مباشر، تتبع التقديمات، استخدام Gmail الشخصي.
- **كيف يعمل:** (توصيل Gmail -> اكتشاف الوظائف -> التقديم بنقرة واحدة).
- **إحصائيات:** عدد الوظائف المتوفرة، التقديمات الناجحة، عدد المستخدمين.

---

### 5.2 المصادقة (Authentication)
**الشاشات:** التسجيل، تسجيل الدخول، نسيت كلمة المرور، التحقق من البريد.

**قواعد العمل:**
- يجب التحقق من البريد قبل الدخول للوحة التحكم.
- إدارة الجلسات عبر JWT (Access Token + Refresh Token).
- قفل الحساب بعد 5 محاولات فاشلة.

---

### 5.3 لوحة التحكم (Dashboard)
**KPI Cards (البطاقات الرئيسية):**
- الوظائف المتاحة
- الوظائف المحفوظة
- معدل الفتح (Open Rate)
- إجمالي البريد المرسل

**الرسوم البيانية:**
- نشاط الإرسال خلال الأسبوع (Sun–Thu).

---

### 5.4 اكتشاف الوظائف (Job Discovery)
**الغرض:** البحث وتصفح الوظائف التي تم جمعها.
- شريط بحث (مسمى وظيفي، تخصص، مدينة).
- تصفية حسب المدن والوقت.
- بطاقات الوظائف تحتوي على: اسم الشركة، المسمى، التاريخ، الموقع، والبريد (نقطة زرقاء).

---

### 5.5 الوظائف المحفوظة (Saved Jobs)
- قائمة شخصية بالوظائف التي ميزها المستخدم.
- خيار "إزالة الكل".

---

### 5.6 التقديم الآلي (Auto-Apply)

#### الحالة 1: Gmail غير متصل
- يظهر تنبيه يمنع الاستخدام ويوجه المستخدم للإعدادات.

#### الحالة 2: Gmail متصل — الخطوة 1 من 2
- اختيار عنوان البريد، نص الرسالة، ورفع السيرة الذاتية.
- قائمة بالوظائف المختارة.

#### الحالة 2: الخطوة 2 — جدولة الإرسال
- تحديد اليوم والوقت للإرسال التلقائي عبر BullMQ.

---

### 5.7 التحليلات والتتبع
**تتبع حالة الطلبات:**
- **قيد الإرسال (In Queue):** الطلب في الطابور.
- **مرسل (Sent):** تم الإرسال بنجاح.
- **فشل (Failed):** فشل الإرسال بعد المحاولات.
- **تم الفتح (Opened):** قام مسؤول التوظيف بفتح البريد.

---

### 5.8 الإعدادات (Settings)
- ربط/فصل حساب Gmail.
- عرض معلومات المنصة والإصدار.

## 6. المتطلبات الوظيفية — لوحة تحكم المسؤول

### 6.2 إدارة الزحف (Scraping Management)
تتحكم في محرك جمع البيانات ومعالجته عبر الذكاء الاصطناعي.

**مصادر الزحف:** تشمل 11 موقعاً سعودياً (Wzayf, Ewdifh, Wdeftksa, etc.).

**خطوات العمل:**
1. سحب HTML من المصدر.
2. استخراج حاويات الوظائف.
3. إرسال HTML الخام إلى LLM API.
4. الحصول على JSON منظم.
5. تخزين البيانات مع فحص الازدواجية.

---

### 6.3 إدارة المستخدمين
- CRUD كامل للمستخدمين (بحث، تصفية، إيقاف، حذف).

---

### 6.4 التنبيهات والإعلانات
- إرسال تنبيهات (نجاح، تحذير، تنبيه) لجميع المستخدمين أو للمشرفين فقط.

---

### 6.5 إعدادات النظام
- وضع الصيانة، تسجيل السجلات المفصل، إعدادات SMTP، وجدولة الزحف.
- **المنطقة الخطيرة:** مسح السجلات، إعادة ضبط المصنع.

## 7. مواصفات نقاط نهاية API

**Base URL:** `https://api.Kafoo.sa/v1`

### 7.1 Auth Endpoints
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `POST` | `/auth/register` | Public | Register a new user |
| `POST` | `/auth/login` | Public | Login and receive tokens |
| `POST` | `/auth/logout` | User | Invalidate refresh token |

### 7.2 Jobs Endpoints
| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| `GET` | `/jobs` | User | Get paginated list of all jobs |
| `POST` | `/jobs/:id/save` | User | Bookmark a job |

**مثال لرد الاستجابة (GET /jobs):**
```json
{
  "data": [
    {
      "id": "uuid",
      "title": "Software Developer",
      "company": "STC",
      "location": "Riyadh",
      "source": "linkedksa",
      "postedAt": "2026-04-01T10:00:00Z",
      "isSaved": false
    }
  ],
  "meta": { "total": 340, "page": 1, "limit": 20 }
}
```

## 8. التكاملات الخارجية

### 8.1 Google Gmail API (OAuth2)
**النطاق المطلوب:** `https://www.googleapis.com/auth/gmail.send`
**الآلية:** يتم تشفير الرموز (Tokens) باستخدام AES-256 وتخزينها في قاعدة البيانات.

### 8.4 BullMQ + Redis (Job Queue)
**الوظيفة:** إدارة طوابير إرسال البريد (فوري أو مجدول) وإعادة المحاولة حتى 3 مرات في حال الفشل.

### 8.5 LLM API (AI-Powered Parsing)
**الدخل:** HTML حاوية الوظيفة.
**الخرج المتوقع (JSON):**
```json
{
  "company_name": "مستشفى دلة",
  "job_titles": ["منسق مركز اتصال", "مشرف خدمات"],
  "application_url": "https://careers.dallahhealth.com/jobs",
  "hr_email": null
}
```

## 9. نماذج البيانات (Data Models)

### 9.1 Users Table
```sql
users (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  full_name       VARCHAR(255) NOT NULL,
  email           VARCHAR(255) UNIQUE NOT NULL,
  role            ENUM('user', 'admin', 'super_admin') DEFAULT 'user',
  status          ENUM('active', 'suspended', 'unverified') DEFAULT 'unverified',
  created_at      TIMESTAMP DEFAULT NOW()
)
```

### 9.3 Jobs Table
```sql
jobs (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  title            VARCHAR(255) NOT NULL,
  company_name     VARCHAR(255) NOT NULL,
  hr_email         VARCHAR(255),
  source           ENUM('wzayf', 'ewdifh', 'linkedksa', ...) NOT NULL,
  posted_at        TIMESTAMP,
  created_at       TIMESTAMP DEFAULT NOW()
)
```

## 13. التقنيات المستخدمة (Technology Stack)

### 13.1 هيكل الـ Monorepo
يستخدم المشروع **Turborepo** مع **pnpm workspaces**.

### 13.2 Backend — `apps/api`
- **اللغة:** TypeScript / Node.js.
- **الإطار:** Express.js.
- **قاعدة البيانات:** PostgreSQL + Prisma ORM.
- **الطوابير:** BullMQ + Redis.
- **الزحف:** Playwright / Cheerio.

### 13.3 Frontend — `apps/web` & `apps/admin`
- **الإطار:** React 18 + Vite.
- **الحالة:** TanStack Query + Zustand.
- **التصميم:** Tailwind CSS + shadcn/ui.
